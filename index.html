Integration & system map

How this site connects to the back office. Breeze (Tithely) is the system of record; the app reads people, events, and roles on a schedule, layers on what Breeze doesn't model (announcements, push, prayer privacy, sermon embeds), and broadcasts through Breeze email/SMS, web push, SoundCloud embeds, and — today, by hand — Facebook + WhatsApp.

The ecosystem & roles map is interactive: every tool grouped by the role it plays, a current-vs-target toggle, click-through detail, and the gaps & tensions to resolve. Event requests

Submitted by members and visitors. Approving pushes the event to Breeze (the system of record); it then appears on the public calendar on the next sync.

Facebook events detected

Posts from the four pages that look like events — structured Facebook Events (high confidence) or dates/times detected in post text (medium). Adding one sends it to the review queue, where a manager still approves it before it reaches Breeze and the calendar. Ignore dismisses it.

Post event to Facebook

Facebook no longer lets apps create Events automatically — the only API path is a gated partner program for ticketing platforms. This prepares everything so a communications person can create the Event as the page in a few clicks, then mark it posted so the team knows it's done.

Share event to WhatsApp

WhatsApp's API can't post into your existing group chats — only a gated program for very high-volume senders can, and not to manually-created groups. This prepares a ready-to-send message so a communications person can drop it into the right WhatsApp group in a couple of taps.
